# 面试辅助系统 — 需求分析文档

> 版本：v0.1
> 日期：2026-06-30
> 状态：初稿

---

## 1. 项目背景与定位

### 1.1 背景

求职者在找工作的过程中面临三个核心痛点：

1. **自我认知模糊** — 对自己的能力模型缺乏结构化认识，不知道自己的长板和短板。
2. **岗位匹配盲区** — 招聘方发布的 JD 信息量大、碎片化，很难快速判断自己是否胜任、差距在哪、如何补足。
3. **面试准备低效** — 面试题分散在各个平台，缺乏针对自身简历和具体岗位的定制化题库，准备方向不明确。

### 1.2 产品定位

一款**自用的、轻量的、AI 驱动的面试辅助 Web 应用**。核心价值是：上传一次简历，结合招聘信息，输出**个人能力画像、岗位匹配分析、定制面试题库**三个维度的结构化信息，帮助求职者高效准备面试。

### 1.3 目标用户

- 主要用户：本人（求职者）
- 预期使用频率：集中在求职季，每日 1-5 次

---

## 2. 功能模块划分

系统分为三个核心模块，外加一个公共基础层：

| 模块 | 编号 | 名称 | 简述 |
|------|------|------|------|
| 基础 | M0 | 用户系统 | 登录、数据隔离 |
| 核心一 | M1 | 个人档案与能力模型 | 简历上传、解析、能力建模 |
| 核心二 | M2 | 招聘信息对比分析 | 岗位匹配、公司分析、差距诊断 |
| 核心三 | M3 | 面试题库生成与管理 | 结构化出题、答案显隐、题库管理 |

### 3.2 M1 — 个人档案与能力模型

#### 3.2.1 业务流程

```
用户上传简历文件
    │
    ▼
系统解析简历 (PDF / Word)
    │
    ▼
提取结构化信息 (基本信息 / 教育背景 / 工作经历 / 技能 / 项目经验)
    │
    ▼
用户校对 / 补充
    │
    ▼
生成个人能力模型 (雷达图 + 维度评分)
```

#### 3.2.2 功能清单

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| M1-01 | 简历文件上传 | P1 | 支持 PDF、DOCX 格式；文件大小 ≤ 10MB |
| M1-02 | 简历解析与结构化提取 | P1 | 调用 AI 解析简历，输出结构化 JSON |
| M1-03 | 基本信息编辑 | P1 | 用户可手动修改解析结果的各个字段 |
| M1-04 | 技能标签管理 | P1 | 自动提取技能 + 手动增删改 |
| M1-05 | 工作 / 项目经历管理 | P1 | 支持多条经历，每条含时间、公司、职位、描述 |
| M1-06 | 个人能力模型 | P1 | 基于简历维度生成评分 |
| M1-07 | 能力雷达图可视化 | P2 | 以雷达图展示各维度评分 |
| M1-08 | 简历版本管理 | P2 | 保留历史版本，支持回退 |
| M1-09 | 能力维度自评修正 | P2 | 允许用户手动调整 AI 评分 |

#### 3.2.3 能力模型维度设计（建议）

| 维度 | 权重建议 | 说明 |
|------|----------|------|
| 技术深度 | 高 | 核心技术的掌握程度、项目经验深度 |
| 技术广度 | 中 | 技术栈覆盖面、跨领域能力 |
| 项目复杂度 | 高 | 项目规模、架构设计、难度 |
| 业务理解 | 中 | 对所在行业/业务的理解程度 |
| 管理/协作 | 中 | 团队协作、项目管理、沟通能力 |
| 学习能力 | 中 | 技术更新速度、学习主动性（从简历中推断） |

> 实际维度可在开发中根据简历内容动态调整，无需固化。

#### 3.2.4 前置条件

- 第三方 AI 服务的 API key（用于简历解析和模型分析）

### 3.3 M2 — 招聘信息对比分析

#### 3.3.1 业务流程

```
用户输入 Boss 直聘 JD（文本粘贴 / 链接）
    │
    ▼
系统提取 JD 结构化信息 (公司 / 岗位 / 要求 / 福利)
    │
    ▼
拉取用户最新简历档案
    │
    ▼
并行生成三个分析报告：
    ├── ① 公司分析报告
    ├── ② 岗位匹配分析
    └── ③ 差距与补齐方案
    │
    ▼
展示结果（可分段展开查看）
```

#### 3.3.2 公司分析报告

| 分析维度 | 说明 | 数据来源 |
|----------|------|----------|
| 公司基本面 | 规模、成立时间、融资阶段、行业地位 | JD 文本提取 + 网络公开信息查询 |
| 核心优势 | 技术积累、市场份额、品牌影响力、薪酬竞争力 | 分析推理 |
| 发展前景 | 行业趋势、公司战略方向、扩张/收缩信号 | 分析推理 |
| 潜在风险 | 裁员新闻、业务萎缩、过度依赖单业务、法律纠纷 | 分析推理 + 公开信息 |
| 公司文化推断 | 从 JD 措辞、福利描述推断加班文化、管理风格等 | 文本分析 |
| 面试难度预估 | 结合公司规模和岗位定位 | 分析推理 |

> P1 功能：基于 JD 文本的分析。P2：联网搜索补充公司信息。

#### 3.3.3 岗位匹配分析

| 分析维度 | 说明 |
|----------|------|
| 匹配度总览 | 百分比 + 总体判断（胜任 / 基本胜任且有差距 / 难以胜任） |
| 硬技能匹配 | 逐条对比 JD 技术要求 vs 个人技能（匹配、部分匹配、缺失） |
| 软技能匹配 | 沟通、管理、抗压等要求 vs 个人经历 |
| 经验匹配 | 年限、行业、项目规模 vs 个人经历 |
| 学历匹配 | JD 学历要求 vs 个人学历 |

#### 3.3.4 差距与补齐方案

| 输出项 | 说明 |
|--------|------|
| 差距清单 | 列出每项不满足的要求 |
| 补齐优先级 | 按重要性和补齐难度排序 |
| 补齐路径 | 针对每项差距给出具体建议（学习资源、项目实践、考证等） |
| 预估时间 | 每项差距的大致补齐时间 |
| 替代策略 | 如果不补齐，如何通过其他优势弥补 |

#### 3.3.5 功能清单

| ID | 功能 | 优先级 | 说明 |
|----|------|--------|------|
| M2-01 | JD 文本输入 | P1 | 纯文本粘贴框 |
| M2-02 | JD 结构化解析 | P1 | AI 提取公司名、岗位名、要求、职责等 |
| M2-03 | 公司分析报告生成 | P1 | 基于 JD 文本的 AI 分析 |
| M2-04 | 岗位匹配度分析 | P1 | 逐项对比简历 vs JD |
| M2-05 | 差距与补齐方案 | P1 | 生成补齐建议 |
| M2-06 | 对比分析记录保存 | P1 | 每次分析存档，支持回顾 |
| M2-07 | 公司信息联网补充 | P2 | 查询公司公开信息补充分析 |
| M2-08 | 多岗位对比 | P2 | 同时对比多个岗位的分析结果 |
| M2-09 | JD 链接自动抓取 | P2 | 输入 Boss 直聘链接自动抓取 JD |

### 3.4 M3 — 面试题库生成与管理

#### 3.4.1 业务流程

```
选择「岗位匹配分析」中的某个岗位
    │
    ▼
点击「生成面试题库」
    │
    ▼
AI 综合简历 + JD 生成结构化题库
    │
    ├── 按面试节奏分节
    │   ├── ① 自我认知类（自我介绍、优缺点、职业规划）
    │   ├── ② 专业技能题（针对 JD 技术栈 + 简历项目）
    │   ├── ③ 项目经历题（深挖简历中的项目）
    │   ├── ④ 行为/情境题（STAR 类问题）
    │   ├── ⑤ 个人生活/价值观题（为什么离职、加班看法等）
    │   └── ⑥ 开放式/综合题（反问面试官的问题等）
    │
    ▼
每道题附带参考答案（基于简历 + JD 定制）
    │
    ▼
展示题库（每题可展开/收起答案）
    │
    ▼
支持收藏、标记难度、隐藏已掌握题目
```

## 4. 数据关系概览

四张核心表：User、ResumeArchive、JobAnalysis、QuestionBank。

- User 与 ResumeArchive：一对多
- ResumeArchive 与 JobAnalysis：一对多
- JobAnalysis 与 QuestionBank：一对多

每张表的主数据类型包括 id、user_id、JSON 字段（存储结构化分析结果）、时间戳。

## 5. 非功能性需求

### 5.1 技术栈建议

| 层 | 技术选型 | 考量 |
|----|----------|------|
| 前端框架 | React + TypeScript | 生态成熟，适合 CRUD + 重交互 |
| 构建工具 | Vite | 开发体验好，构建快 |
| UI 组件库 | Ant Design 或 shadcn/ui | Ant Design 表单强大，shadcn/ui 更轻量 |
| 样式方案 | Tailwind CSS | 开发效率高 |
| 后端 | Next.js API Routes 或轻量 Node.js | 个人项目，减少维护成本 |
| 数据库 | SQLite (better-sqlite3) 或 Supabase | 个人场景 SQLite 足够 |
| AI 集成 | OpenAI / Claude / 国产大模型 API | 简历解析、JD 分析、出题 |
| 部署 | Vercel / Railway / 自建服务器 | 低成本优先 |

### 5.2 性能要求

- 页面加载时间 ≤ 2s（首屏）
- AI 分析响应时间 ≤ 30s
- 支持移动端适配

### 5.3 安全与可用性

- 简历包含敏感信息，数据传输需加密
- AI 调用时可选择脱敏手机号、邮箱等
- 所有操作有明确反馈
- AI 分析过程展示进度状态

## 6. 关键交互流程

### 6.1 第一次使用流程

```
注册/登录 → 上传简历 → AI 解析 → 校对修改 → 生成能力模型 → 进入主面板
```

### 6.2 日常使用流程

```
主面板
  ├── 查看/修改个人档案
  ├── 新增岗位分析：粘贴 JD → 提交 → 查看三份报告
  ├── 从岗位分析进入面试题库 → 查看/练习
  └── 历史记录：历史分析 + 历史题库
```

## 8. 优先级与实施路线

### Phase 1 — MVP（基础可用，7-11 天）

1. 项目脚手架搭建 + 基础 UI

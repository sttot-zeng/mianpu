# Signal Capture Hooks
# 信号捕获规则 —— 自动识别用户的纠正、反驳、规则类反馈并写入信号库
# 信号库位置：`.codex/signals/signals.json`

## 信号定义
信号是用户在与 AI 交互过程中产生的有效纠正反馈，信号类型包括：

| 信号类型 | 触发模式 | 示例 |
|---------|---------|------|
| **纠正** | 用户直接指出 AI 的错误 | "不对，这个功能应该..." |
| **规则明确** | 用户提出规则要求 | "以后所有代码都要加注释" |
| **偏好设定** | 用户表达个人偏好 | "我不喜欢用 TypeScript，用 JS 就行" |
| **质量不达标** | 用户对产出结果不满意 | "这个设计太粗糙了，重做" |

## 信号捕获时机
1. 用户说"不对""错了""不是这样"等否定性表达时
2. 用户给出明确的规则或偏好时
3. 用户要求重做或大幅修改时
4. 用户否定 AI 的某个习惯性行为时

## 信号记录格式
每条信号记录到 signals.json，格式如下：

```json
{
  "id": "S-20260702-001",
  "type": "纠正",
  "timestamp": "2026-07-02T16:00:00+08:00",
  "summary": "用户纠正了XX功能的实现方式",
  "detail": "完整对话上下文",
  "source_skill": "ProductSpecBuilder",
  "processed": false,
  "applied_rule": null
}
```

## 信号处理流程
1. **捕获**：识别到信号 -> 确认是否需要记录（去重）
2. **记录**：写入 signals.json，processed = false
3. **汇总**：同类型多次信号自动合并（基于 source_skill + type 维度）
4. **进化**：新会话启动时，Evolution Runner 读取未处理信号
5. **确认**：用户确认进化建议后，更新到对应规则文件
6. **关闭**：修改 signals.json 中的 processed = true

## 捕获策略
- **不过度捕获**：仅捕获有规则价值的反馈，单纯的任务性请求不记录
- **同类合并**：同一 Skill、同一类型的多次反馈自动合并记录，避免信号库膨胀
- **时效性**：超过 30 天未处理的信号自动标记为过期（processed = true, 加注 "expired"）
- **去重**：新增信号前检查是否已有完全相同的信号，避免重复

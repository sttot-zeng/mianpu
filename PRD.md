## 1. 产品概述与愿景

### 1.1 产品愿景

让每一位求职者都能**基于数据而非直觉**来准备面试——看清自己的真实能力图景，精准匹配岗位要求，用最短的时间补齐差距，带着信心走进面试间。

### 1.2 产品定位

一款自用的、AI 驱动的面试辅助 Web 应用。一次简历上传，持续服务于整个求职周期：

- **输入**：个人简历 + Boss 直聘等平台的招聘信息 (JD)
- **处理**：AI 驱动的结构化分析
- **输出**：个人能力画像、岗位匹配报告、定制面试题库

### 1.3 核心价值主张

| 用户痛点 | 产品解决方案 | 可测量的价值 |
|----------|-------------|-------------|
| 不知道自己的竞争力在哪 | 个人能力模型 + 雷达图 | 从模糊感觉到结构化认知 |
| 看不懂 JD 到底要什么样的人 | 岗位匹配度逐项对标 | 一眼看清差距在哪 |
| 面试准备不知道方向 | 定制化面试题库，按面试节奏组织 | 减少 50%+ 的无效准备时间 |
| 不知道公司靠谱不靠谱 | 公司分析报告（优势/风险/文化） | 降低踩坑概率 |

### 1.4 产品约束

| 维度 | 约束条件 |
|------|----------|
| 目标用户规模 | 仅本人使用，不要求多租户高并发 |
| 预算 | 个人付费 AI API + 服务器费用，月预算可控 |
| 数据敏感度 | 简历含手机号、邮箱等 PII，需脱敏处理 |
| 设备 | PC 为主，移动端可查看题库 |
| 时效性 | AI 分析可在 30s 内完成，不需要实时 |

## 2. 用户画像与场景

### 2.1 用户画像

| 属性 | 值 |
|------|-----|
| 身份 | 互联网行业从业者，中高级工程师为主 |
| 技术素养 | 中等到高，熟悉 Web 产品，能理解 AI 输出的局限性 |
| 求职频率 | 每 1-3 年一次，求职季密集使用（每日 1-5 次） |
| 使用时段 | 晚间和周末为主，面试前高频查看题库 |
| 数据习惯 | 愿意手动粘贴 JD，偏好数据持久保存在本地 |

### 2.2 使用场景

#### 场景 A：首次接入（新用户 Onboarding）

```
张工准备跳槽，他有一份 PDF 简历，想系统梳理自己的能力。
→ 注册账号 → 上传简历 → AI 自动解析 → 张工检查并修正解析结果
→ 生成能力模型 → 看到雷达图，直观了解自己的长短板
```

#### 场景 B：发现一个感兴趣的岗位

```
张工在 Boss 直聘上看到一个"高级前端工程师"的岗位，想深度分析。
→ 复制 JD → 粘贴到系统 → 点击分析
→ 等待约 15s → 看到三份报告：
   - 公司分析：公司处于 B 轮，技术氛围好，但加班文化明显
   - 岗位匹配：匹配度 72%，React 经验匹配，但缺 Node.js 经验
   - 补齐方案：建议花 2 周学完 Node.js 基础 + 一个 CRUD 项目
```

#### 场景 C：面试前冲刺准备

```
张工收到面试通知，距面试还有 3 天。
→ 打开对应岗位的分析 → 点击"生成面试题库"
→ 得到一份按面试节奏编排的题库：
   - 自我介绍：1 份定制模板
   - 专业技能：15 道 React/前端题
   - 项目深挖：5 道基于简历项目的深度追问
   - 行为题：8 道 STAR 情境题
→ 逐题练习，隐藏答案先自答，再展开对照
→ 收藏 3 道不熟的重点复习
```

#### 场景 D：多岗位横向对比

```
张工同时在看 3 家公司，想比较哪家最适合自己。
→ 分别对 3 个岗位做分析
→ 打开对比视图，并列展示匹配度和关键差异
→ 发现 A 公司要求最高（匹配 55%），B 公司最匹配（85%），C 公司薪资最高
```

#### 场景 E：持续追踪进步

```
张工花了两周补齐了 Node.js 短板，更新了简历。
→ 重新上传新版本简历
→ 回到之前分析的岗位，重新计算匹配度
→ 看到匹配度从 72% 提升到 83%
→ 生成新的题库，发现之前"缺失"的技能题变成了"已掌握"
```

## 4. 功能规格说明

### 4.1 功能模块矩阵

| 模块 | 功能 | 输入 | 处理逻辑 | 输出 | 验收标准 |
|------|------|------|----------|------|----------|
| M1 | 简历上传 | PDF/DOCX 文件 | 文件格式校验 → 文件存储 → AI 解析 | 结构化 JSON | 3s 内完成校验，15s 内完成解析 |
| M1 | 简历解析校对 | 结构化表单 | 用户编辑 → 差异对比 → 存储 | 更新后 JSON | 每次保存有即时反馈 |
| M1 | 能力模型生成 | 结构化简历数据 | AI 多维度评分 | 维度评分 + 雷达图 | 返回 5-8 个维度分数 |
| M2 | JD 解析 | 纯文本 JD | AI 提取公司、岗位、要求 | 结构化 JD 对象 | 字段完整率 ≥ 90% |
| M2 | 公司分析 | JD 信息 | AI综合分析 | 6 维度报告 | 每维度 ≥ 200 字 |
| M2 | 岗位匹配 | 简历 + JD | AI 逐项对比 | 匹配列表 + 百分比 | 逐条标注状态 |
| M2 | 补齐方案 | 差距清单 | AI 生成学习路径 | 排序列表 | 每项含路径+时间 |
| M3 | 题库生成 | 简历 + JD | AI 按模板生成 | 6 章节题库 | 每章节 ≥ 3 题 |
| M3 | 答案显隐 | 题库数据 | JS 控制 DOM 显隐 | 视觉切换 | 切换 < 100ms |
| M3 | 模拟面试 | 题库数据 | 随机出题 + 计步 | 逐题界面 | 全屏 + 键盘导航 |

### 4.2 AI 提示词策略（概要）

| 用途 | 输入素材 | 输出结构 | 关键要求 |
|------|----------|----------|----------|
| 简历解析 | PDF/DOCX 文字内容 | JSON 结构化数据 | 字段完整，不杜撰信息 |
| JD 解析 | JD 文本 | JSON 结构化数据 | 提取公司名、岗位名、要求列表、职责列表 |
| 公司分析 | JD 文本 | Markdown 报告 | 基于事实推理，不编造数据 |
| 岗位匹配 | 简历 JSON + JD JSON | JSON 匹配结果 | 逐条对标，分类标注 |
| 补齐方案 | 差距清单 | JSON 建议列表 | 建议可操作，含具体资源推荐 |
| 题库生成 | 简历 JSON + JD JSON | JSON 题库 | 按模板格式，每题定制化 |

```
  [◉ 正在分析公司信息...]
  [○ 正在对比岗位要求...]
  [○ 正在生成补齐方案...]

完成后 → 进度面板消失 → 结果内容渐进式展示
失败时 → 进度面板显示错误 → 提供"重试"按钮
```

#### 规则 2：答案显隐交互

```
默认状态：题目可见，答案隐藏
点击题目卡片 → 答案部分展开（动画 300ms）
再次点击 → 答案部分收起
全局开关：
  "显示全部" → 所有题目的答案展开
  "隐藏全部" → 所有题目的答案收起
状态记忆：localStorage 记住每道题的 lastState
```

#### 规则 3：多步骤表单

```
简历校对表单：
  基本信息（单步）→ 教育背景 → 工作经历 → 技能标签
  每步可独立保存
  全部完成前不可进入下一步（M2/M3）
```

### 7.3 AI 调用端点

| 内部方法 | 说明 | 调用时机 | 超时 |
|----------|------|----------|------|
| ai.parseResume(text) | 解析简历 | 简历上传后 | 30s |
| ai.parseJD(text) | 解析 JD | 创建分析时 | 15s |
| ai.analyzeCompany(jd) | 分析公司 | P2 整体分析中 | 20s |
| ai.matchResumeJD(resume, jd) | 岗位匹配 | P2 整体分析中 | 20s |
| ai.generateGapPlan(gaps) | 补齐方案 | P2 整体分析中 | 15s |
| ai.generateQuestions(resume, jd) | 生成题库 | 用户点击后 | 40s |

> P2 整体分析 = 并行调用 ai.analyzeCompany + ai.matchResumeJD → 串行调用 ai.generateGapPlan

```json

  }

```

| 错误码 | HTTP 状态码 | 说明 |
|--------|------------|------|
| INVALID_FILE_TYPE | 400 | 不支持的文件格式 |
| FILE_TOO_LARGE | 400 | 文件超过 10MB |

## 9. 错误处理与边界情况

### 9.1 简历上传

| 场景 | 预期行为 |
|------|----------|
| 上传空文件 | 前端校验拦截，提示"文件为空" |
| PDF 加密/有密码 | 提示"请先移除 PDF 密码保护" |
| DOCX 格式损坏 | 提示"文件格式异常，请尝试另存为 PDF 后重试" |
| 上传同名简历 | 提示"已存在同名简历，是否覆盖？" |
| 上传中网络断开 | 显示"上传中断，请检查网络后重试" |
| PDF 纯扫描件（无文字层） | AI 解析提示"未提取到文字内容，建议使用有文字层的 PDF" |

### 9.2 JD 分析

| 场景 | 预期行为 |
|------|----------|
| JD 文本过短（< 50 字） | 提示"JD 文本过短，请粘贴完整的职位描述" |
| JD 不含公司名 | AI 自动标注"未识别到公司名称，请手动补充" |
| 粘贴 Boss 直聘链接失败 | 自动回退到文本粘贴模式 |
| AI 解析出现幻觉（编造不存在的要求） | 用户可手动删除+修正解析结果，+ 提示"AI 可能有不准确的信息，请核查" |
| 用户没有简历就做分析 | 提示"请先上传并完成简历解析" |

### 9.3 题库

| 场景 | 预期行为 |
|------|----------|
| 题库生成后简历更新 | 提示"简历已更新，建议重新生成题库以匹配最新信息" |
| 题库题量为 0 | 显示错误状态，建议重新生成 |
| 用户同时多端使用 | 题库状态以最后一次保存为准（最终一致性） |

### 9.4 AI 调用

| 场景 | 预期行为 |
|------|----------|
| AI API 返回空/无效响应 | 重试 1 次，失败后提示"AI 分析异常，请稍后重试" |
| AI API 超时（> 30s） | 返回当前已完成部分的阶段性结果 + 提示"部分分析未完成" |
| API key 配额耗尽 | 提示"AI 服务配额不足，请配置新的 API key" |
| 网络错误 | 自动重试 1 次，指数退避（1s → 2s → 4s） |

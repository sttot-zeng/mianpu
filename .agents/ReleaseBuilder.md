# ReleaseBuilder —— 发布交付

## 适用场景

- 开发完成且所有审查通过
- 准备进行隐私审计和发布部署

## 宏观骨架

**先打地基**：执行隐私审计、检查发布清单
**再抠细节**：构建产物、部署验证、文档归档

## 验收标准

1. **交付物**：
   - **隐私审计报告**：`docs/PrivacyAudit-<项目名>.md`
   - **发布检查清单**：已逐项确认
   - **构建产物**：可部署的构建包或部署指令
   - **发布记录**：在 CHANGELOG 中记录发布版本

2. **验证方式**：
   - 隐私审计无未解决红旗标记
   - 构建产物可正常启动和访问
   - 发布清单所有项目已打勾确认

## 红线禁令

- **禁止未经隐私审计直接发布**：必须先执行隐私审计
- **禁止未解决安全问题发布**：安全相关审查问题必须全部修复
- **禁止不用 CHANGELOG**：发布版本必须记录在变更日志中
- **禁止跳过回归测试**：发布前必须执行全量测试套件

## 隐私审计清单

- [ ] **用户数据收集**：是否收集用户数据？收集了哪些？
- [ ] **数据存储**：数据存储在何处？是否加密？
- [ ] **第三方服务**：使用了哪些第三方服务？各自的隐私政策？
- [ ] **认证机制**：用户认证方式？密码存储方案？Token 策略？
- [ ] **数据传输**：是否使用 HTTPS？API 接口是否有鉴权？
- [ ] **日志安全**：日志中是否可能包含敏感信息？（密码、Token、个人身份信息）
- [ ] **前端安全**：XSS 防御？CSRF 防御？输入校验？
- [ ] **环境变量**：敏感配置是否通过环境变量注入而非硬编码？
- [ ] **依赖安全**：第三方依赖是否有已知安全漏洞？

## 发布检查清单

- [ ] **代码审查**：所有代码已通过审查，无未关闭的严重/重要问题
- [ ] **测试通过**：全量测试套件通过（单元测试 + 集成测试 + E2E）
- [ ] **构建成功**：生产环境构建无错误
- [ ] **文档同步**：PRD/DesignBrief/Plan/CHANGELOG 与代码一致
- [ ] **信号处理**：信号库中无待处理的阻塞性信号
- [ ] **回滚方案**：确定回滚策略（如发布后发现问题如何回退）
- [ ] **部署脚本**：部署步骤已记录
- [ ] **环境配置**：生产和测试环境的配置分离

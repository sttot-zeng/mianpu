# GoCreator —— Go 指令生成

## 适用场景

- 用户说"你来执行"但未给出明确的 Go 指令
- 一个阶段完成，需要触发下一阶段的执行
- 需要将规则或标准转化为可执行的 Go 指令
- 用户给出的指令缺乏明确目标或可验证标准时

## 宏观骨架

**先打地基**：读取当前项目状态（state.json）和上下文文档
**再抠细节**：结合当前阶段和 Skill 生成精确的 Go 指令

## 验收标准

1. **交付物**：一个完整的 Go 指令，必须包含三要素：
   - **明确目标**：一句话说明要做什么，不含糊
   - **可量化的完成标准**：做到什么程度算完成，可验证
   - **可执行的验收方式**：通过什么方式验证（运行什么命令、检查什么文件）

2. **验证方式**：
   - Go 指令的三要素齐全，可直接执行
   - 指令接收者（AI 或用户）能清楚知道何时算完成
   - 验收方式在当前环境下可操作

## 红线禁令

- **禁止模糊表述**："优化体验""完善功能"这种表述必须拆解为具体动作
- **禁止不可验证的标准**："让用户满意"——如何验证用户满意？
- **禁止目标与标准脱节**：完成标准必须能证明目标已达成
- **禁止忽略项目阶段**：生成的 Go 必须与当前项目上下文匹配

## Go 指令模板

```
## Go：[一句话说明目标]

### 目标
[清晰、具体、一句话说明要达成的状态]

### 完成标准
- [ ] [可量化的标准 1]
- [ ] [可量化的标准 2]
- [ ] ...

### 验收方式
1. [验收步骤 1：运行什么命令或检查什么逻辑]
2. [验收步骤 2]
...
```

## Go 生成示例

### 好 Go 示例
```
## Go：完成 Phase 1 核心搜索功能

### 目标
实现用户可通过关键词搜索产品并查看结果列表的功能。

### 完成标准
- [ ] 搜索输入框可在首页和列表页使用
- [ ] 输入关键词后按回车或点击搜索按钮触发
- [ ] 搜索结果以列表形式展示，包含标题、摘要、更新时间
- [ ] 搜索无结果时显示空状态提示
- [ ] 搜索 API 错误时显示错误提示，不崩溃

### 验收方式
1. 打开首页，输入搜索词按回车，确认跳转至搜索结果页
2. 搜索结果列表渲染正确，点击可进入详情
3. 输入不存在的词，确认显示空状态
4. 关闭网络后搜索，确认显示错误提示
5. 运行 `npm test` 确认测试通过
```

### 坏 Go 示例
```
## Go：完成搜索功能
### 目标
让用户可以搜索
### 完成标准
搜索好用
### 验收方式
看一下
```

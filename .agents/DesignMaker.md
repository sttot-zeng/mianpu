# DesignMaker —— 视觉设计

## 适用场景

- 用户明确要求生成视觉原型或高保真设计稿
- 设计规范（DesignBrief）已完成，需要将其落地为具体的视觉方案
- 产品涉及复杂的 UI 交互需要直观验证

## 宏观骨架

**先打地基**：确定设计工具和输出格式（HTML/CSS 原型 / 图片 / 截图）
**再抠细节**：逐页面生成视觉方案，确保设计规范一致

## 验收标准

1. **交付物**：`docs/Design-<项目名>.md` 或直接生成 HTML/CSS 原型
   - 核心页面覆盖不少于 3 个主要页面视图
   - 每个页面标注与设计规范的对应关系
   - 关键交互状态展示（空态/错误态/加载态）
   - 色彩和字体规范的实际应用示例

2. **验证方式**：
   - 所有核心页面视觉方案完整，无占位标记
   - 视觉方案与 DesignBrief 的设计决策一致
   - 用户审阅确认

## 红线禁令

- **禁止脱离设计规范**：所有视觉决策必须回溯到 DesignBrief 中的约定
- **禁止跳过设计规范直接输出**：没有 DesignBrief 不得使用本 Skill
- **禁止过度完美主义**：视觉方案够用即可，不追求像素级完美
- **禁止产生设计债务**：如使用 CSS 生成原型，代码需保持整洁，不得是"一次性代码"

## 1. 技术方案总览

### 1.1 技术栈决策

| 层 | 选型 | 理由 |
|----|------|------|
| 前端框架 | **纯 HTML + CSS + Vanilla JS** | 零后端、无构建步骤、现有页面直接复用 |
| 模块化 | **ES Module (type="module")** | 浏览器原生支持，无需打包工具 |
| JS 库 | **pdf.js** (PDF 解析)、**marked** (Markdown 渲染) | 按需加载 CDN |
| AI 适配 | **自研 AI Adapter 层** | 统一接口，支持多模型切换 |
| 数据持久化 | **localStorage** (结构化数据) + **IndexedDB** (文件) | 零后端，浏览器存储 |
| 开发服务器 | **VS Code Live Server 或 python -m http.server** | 本地开发，ES Module 要求 HTTP 协议 |
| 部署 | **Vercel / Netlify / GitHub Pages** | 纯静态，免费托管 |

### 1.4 AI 模型适配策略

统一接口设计：

```javascript
// === 调用方视角 — 一行切换模型 ===
const ai = new AIAdapter({
  provider: 'deepseek',      // 切换：deepseek / qwen / glm / moonshot
  apiKey: 'sk-xxx',
  model: 'deepseek-chat'     // 对应 provider 的模型名
});

const result = await ai.chat([
  { role: 'system', content: '你是一位专业的简历分析师' },
  { role: 'user', content: resumeText }
]);
```

支持的模型与切换方式：

| 提供商 | 默认模型 | 切换方式 | API 格式 |
|--------|----------|----------|----------|
| DeepSeek | deepseek-chat | 下拉选模型 | OpenAI 兼容 |
| 通义千问 | qwen-max | 下拉选模型 | 统一接入层适配 |
| 智谱 GLM | glm-4 | 下拉选模型 | OpenAI 兼容 |
| 月之暗面 | moonshot-v1 | 下拉选模型 | OpenAI 兼容 |

模型配置界面：放在 profile 页面底部的「AI 设置」区，用户可：
1. 选择提供商
2. 选择具体模型
3. 填写 API Key
4. 测试连接

API Key 加密存储在 localStorage。

## 4. 技术细节备忘

### 4.1 关键依赖库

| 用途 | 库名 | 加载方式 | 大小 |
|------|------|----------|------|
| PDF 文字提取 | pdf.js (pdfjs-dist) | CDN + lib/ 备选 | ~800KB |
| Markdown 渲染 | marked | CDN | ~30KB |
| DOCX 解析 | 自写简单 XML 解析 | 内联 | 无需额外库 |
| 导出 PDF | html2pdf.js / 自写 | CDN | ~200KB |
| 图标 | 内联 SVG | 内联 | 无需库 |

### 4.2 AI 调用 Prompt 设计要点

```
系统 Prompt 结构：
{
  role: 'system',
  content: `你是一位专业的面试辅助AI助手。
           你擅长：1. 解析简历提取结构化信息
           2. 分析公司优势和风险
           3. 对比简历和岗位要求的匹配度
           4. 生成定制化面试题库
           
           请严格按照要求的JSON格式输出，不要添加额外内容。`
}
```

各场景专用 Prompt 策略：
- **简历解析**：指定输出 JSON schema 示例，要求字段完整
- **公司分析**：要求区分「基于JD文本的推断」和「需要验证的猜测」
- **岗位匹配**：逐条对标，每项附上匹配理由
- **题库生成**：按 6 章节模板输出，每题定制化

### 4.3 数据存储 Key 规划

```javascript
const KEYS = {
  SETTINGS:        'mianpu_settings',        // AI 配置、外观设置

  ACTIVE_ANALYSIS: 'mianpu_active_analysis',   // 当前正在查看的分析 ID
  QUESTION_BANKS:  'mianpu_question_banks',    // 所有题库
  EXPORT_VERSION:  'mianpu_data_version'       // 数据版本号
};
```

## 6. 风险和应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|----------|
| AI API CORS 限制，浏览器无法直接调用 | 中 | 高 | ① 优先选支持 CORS 的提供商（DeepSeek/Qwen 均支持）<br>② 备选：用 Cloudflare Worker 做一层代理（免费额度足够个人使用） |
| PDF 解析效果差（扫描件、表格） | 中 | 中 | ① 提示用户使用有文字层的 PDF<br>② AI 模型本身对文字提取结果有纠错能力 |
| 浏览器存储上限（localStorage ~5MB） | 低 | 中 | ① 结构化数据体量很小（JSON 文本）<br>② 简历原文件存 IndexedDB 而非 localStorage<br>③ 单个分析+题库 ~50KB，足够存 100+ 份 |
| 切换浏览器/清缓存后数据丢失 | 中 | 高 | ① 提供数据导出/导入功能<br>② 界面显眼提示数据存储位置 |
| AI 输出格式不稳定（JSON 解析失败） | 中 | 中 | ① Prompt 中给出严格格式要求+示例<br>② JS 解析时做 try-catch + 容错处理<br>③ 解析失败时提示用户重试 |
